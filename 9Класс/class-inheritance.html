Наследование классов

Было бы глупо если классы были бы, а наследования не было. Интерфейсов в чистом js нет. Множественное наследование не возможно.

Ключевое слово «extends»

Как и в Java - наследование осуществляется при помощи слова extends. С тем отличие, что после extends разрешены любые выражения: 

        
function f(phrase) {
  return class {
    sayHi() { alert(phrase); }
  };
}

class User extends f("Привет") {}

new User().sayHi(); // Привет

Переопределение методов

Инструкции @Override в языке нет, есть в TS, но это другая история.

Методы переопределяются автоматически, вызывается метод потомка если это объект потомка.

Как и в Java есть ключевое слово super:

  • super.method() вызывает родительский метод.
  • super() вызывает родительский конструктор
У стрелочных функции нет родителя, а потому вызывается super внешнего контекста: 
        
class Rabbit extends Animal {
  stop() {
    setTimeout(() => super.stop(), 1000); // вызывает родительский stop после 1 секунды
  }
}

Переопределение конструктора

Если в объекте конструктора нет, то происходит автоматическая подстановка: 

    
class Rabbit extends Animal {
  // генерируется для классов-потомков, у которых нет своего конструктора
  constructor(...args) {
    super(...args);
  }
}

Если он есть, то сначала надо явно вызвать конструктор родителя, а потом уже обращаться к this. 

    
class Animal {

  constructor(name) {
    this.speed = 0;
    this.name = name;
  }

  // ...
}

class Rabbit extends Animal {

  constructor(name, earLength) {
    super(name);
    this.earLength = earLength;
  }

  // ...
}

// теперь работает
let rabbit = new Rabbit("Белый кролик", 10);
alert(rabbit.name); // Белый кролик
alert(rabbit.earLength); // 10
По другому не сработает.

Переопределение полей класса: тонкое замечание

            
class Animal {
  name = 'animal';

  constructor() {
    alert(this.name); // (*)
  }
}

class Rabbit extends Animal {
  name = 'rabbit';
}

new Animal(); // animal
new Rabbit(); // animal
Но вот методы класса всегда вызываются наследника, если оно у него есть.

Устройство super, [[HomeObject]]

Скипаю этот раздел, по причине его ненужности для разработки, тут обсуждается внутренняя кухня языка

Итого

  1. Чтобы унаследовать от класса: class Child extends Parent:
    При этом Child.prototype.__proto__ будет равен Parent.prototype, так что методы будут унаследованы.
  2. При переопределении конструктора:
    Обязателен вызов конструктора родителя super() в конструкторе Child до обращения к this.
  3. При переопределении другого метода:
    Мы можем вызвать super.method() в методе Child для обращения к методу родителя Parent.
  4. Внутренние детали:
    Методы запоминают свой объект во внутреннем свойстве [[HomeObject]]. Благодаря этому работает super, он в его прототипе ищет родительские методы.
    Поэтому копировать метод, использующий super, между разными объектами небезопасно.

У стрелочных функций нет своего this и super, поэтому они «прозрачно» встраиваются во внешний контекст.
