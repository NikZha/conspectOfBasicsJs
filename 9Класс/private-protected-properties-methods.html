Приватные и защищённые методы и свойства

Большая часть топика просто скипнута, потому что там пример, там какой-то не нужный мне текст из рассуждений.

В целом можно выделить три варианта private в JavaScript: неявное соглашение, только чтение и настоящая приватность.

Защищённое свойство

Первый вариант: 


            
        class TestClass {

            constructor(parameters) {
            this._fakePrivate = parameters    
            }

        }
        const obj = new TestClass('test')
        function fakePrivate(params) {
            alert(obj._fakePrivate)
            obj._fakePrivate = "fake"
            alert(obj._fakePrivate)
        }
Как видно это всего лишь договорённость которая фактически не защищает ни от чего.

Свойство только для чтения

Второй вариант, основан на get/set 


            
        class TestClass1 {
            constructor(parameters) {
                this._onlyRead = parameters;
            }
            get onlyRead(){
                return this._onlyRead;
            }
        }

        function testR(params) {
            const obj = new TestClass1("Test only read prop")
            alert(obj.onlyRead)
            obj._onlyRead = "test"
            alert(obj.onlyRead)
            obj.onlyRead = "111"
            alert(obj.onlyRead)
        }
В целом он получше, но доступ к внутреннему состоянию так же есть.

Приватное свойство

Собственно это и есть по настоящему приватное свойство, в которое нельзя писать как попало: 


            
        class TestClass2 {
            #privateProp = null
            constructor(parameters) {
                this.#privateProp = parameters
             }
             get privateProp(){
                return this.#privateProp;
             }
        }
        function testPrivate(params) {
            const obj = new TestClass2("PRIVATE")
            alert(obj.privateProp)
            obj.privateProp = "NON"
            alert(obj.privateProp)
          //  obj.#privateProp = "NON" Mistake!
            alert(obj.privateProp)
        }
Так же следует отметить, что this['#name'] не работает. Это ограничение синтаксиса сделано для обеспечения приватности.

Итого

Единственная приватность это #name для поля и метода. Все остальное фикция и надежда на то что ты сам и другие разработчики будут соблюдать эти шаткие договорённости.

В приватных полях и методах есть одна тонкость - приватные поля и методы не наследуются и не сериализуются посредством JSON и эту особенность надо серьёзно учитывать.

Никакого protected в JavaScript нет.
