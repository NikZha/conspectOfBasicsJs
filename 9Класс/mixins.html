Примеси

Примеси это вариант множественного наследования, но через композицию, когда методы другого объекта добавляются путём вызова Object.assign()в текущий объект и могут наследоваться от текущего объекта.

Класс-примесь не используют напрямую, только, как донора методов.

Пример примеси


        
            // примесь
            let sayHiMixin = {
                sayHi() {
                    alert(`Привет, ${this.name}`);
                },
                sayBye() {
                    alert(`Пока, ${this.name}`);
                }
            };

            // использование:
            class User {
                constructor(name) {
                    this.name = name;
                }
            }

            // копируем методы
            Object.assign(User.prototype, sayHiMixin);

            // теперь User может сказать Привет
            new User("Вася").sayHi(); // Привет, Вася!

            class PowerUser extends User {
                constructor(parameters) {
                    super(parameters)
                }
            }
            const admin = new PowerUser("Петя")
            admin.sayBye()
Как видно из этого примера, из объекта добавляются методы и так же эти методы наследуются.

Примеси могут собирать методы путём наследования: 

    
let sayMixin = {
  say(phrase) {
    alert(phrase);
  }
};

let sayHiMixin = {
  __proto__: sayMixin, // (или мы можем использовать Object.setPrototypeOf для задания прототипа)

  sayHi() {
    // вызываем метод родителя
    super.say(`Привет, ${this.name}`); // (*)
  },
  sayBye() {
    super.say(`Пока, ${this.name}`); // (*)
  }
};

class User {
  constructor(name) {
    this.name = name;
  }
}

// копируем методы
Object.assign(User.prototype, sayHiMixin);

// теперь User может сказать Привет

Итого

Примесь – общий термин в объектно-ориентированном программировании: класс, который содержит в себе методы для других классов.
Некоторые другие языки допускают множественное наследование. JavaScript не поддерживает множественное наследование, но с помощью примесей мы можем реализовать нечто похожее, скопировав методы в прототип.
Мы можем использовать примеси для расширения функциональности классов, например, для обработки событий, как мы сделали это выше.
С примесями могут возникнуть конфликты, если они перезаписывают существующие методы класса. Стоит помнить об этом и быть внимательнее при выборе имён для методов примеси, чтобы их избежать.
