Промисы

Синтаксис создания Promise: 

                
let promise = new Promise(function(resolve, reject) {
  // функция-исполнитель (executor)
  // "певец"
});
Где, callback'и:
  • resolve(value) — если работа завершилась успешно, с результатом value.
  • reject(error) — если произошла ошибка, error – объект ошибки. 
    
    let promise = new Promise(function(resolve, reject) {
  // эта функция выполнится автоматически, при вызове new Promise

  // через 1 секунду сигнализировать, что задача выполнена с результатом "done"
  setTimeout(() => resolve("done"), 1000);
});
function resolve(params) {
    alert(params)
}
promise.then(
    result => resolve(result)
)
Я чуть забежал вперёд, но чтобы было очевидно, что промисы работают с последующим вызовом then, catch, finally. А сам по себе объект или функция результат не возвращают. Есть ещё такой момент что если какая-то функция выполнится сразу, то промис вернёт первый ответ и не станет дожидаться оставщиеся.

Потребители: then, catch

then

Синтаксис: 

    
promise.then(
  function(result) { /* обработает успешное выполнение */ },
  function(error) { /* обработает ошибку */ }
);
Более наглядный пример: 
    
let promise = new Promise(function(resolve, reject) {
  setTimeout(() => resolve("done!"), 1000);
});

// resolve запустит первую функцию, переданную в .then
promise.then(
  result => alert(result), // выведет "done!" через одну секунду
  error => alert(error) // не будет запущена
);
То есть в случае штатного завершения отработает result, в случае ошибки error.

Если объект ошибки не важен, как и обработка ошибок, можно просто: 

    
let promise = new Promise(resolve => {
  setTimeout(() => resolve("done!"), 1000);
});

promise.then(alert); // выведет "done!" спустя одну секунду

catch

Как и в случае с последним вариантом, можно обработать только ошибку: 

    
let promise = new Promise((resolve, reject) => {
  setTimeout(() => reject(new Error("Ошибка!")), 1000);
});

// .catch(f) это то же самое, что promise.then(null, f)
promise.catch(alert); // выведет "Error: Ошибка!" спустя одну секунду

Очистка: finally

Как и в случае обработки ошибок выполнится finally всегда и вперёд возврата результата: 


    
new Promise((resolve, reject) => {
  setTimeout(() => resolve("value"), 2000);
})
  .finally(() => alert("Промис завершён")) // срабатывает первым
  .then(result => alert(result)); // <-- .then показывает "value"
Подведем итог:
  • Обработчик finally не получает результат предыдущего обработчика (у него нет аргументов). Вместо этого этот результат передается следующему подходящему обработчику.
  • Если обработчик finally возвращает что-то, это игнорируется.
  • Когда finally выдает ошибку, выполнение переходит к ближайшему обработчику ошибок.

Пример: loadScript

Как видно из этого примера, автор упустил один большой нюанс: 


            
function loadScript(src) {
  return new Promise(function(resolve, reject) {
    let script = document.createElement('script');
    script.src = src;

    script.onload = () => resolve(script);
    script.onerror = () => reject(new Error(`Ошибка загрузки скрипта ${src}`));

    document.head.append(script);
  });
}

let promise = loadScript("https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.11/lodash.js");

promise.then(
  script => alert(`${script.src} загружен!`),
  error => alert(`Ошибка: ${error.message}`)
);

promise.then(script => alert('Ещё один обработчик...'));
.then можно выполнять несколько раз передавая результат одной или разным функциям.

Второй особенностью является то, что результат можно получить в любом месте программы.
