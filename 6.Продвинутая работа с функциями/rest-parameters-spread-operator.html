Остаточные параметры и оператор расширения

Раздел о:

  • fun(a, b, ..args)
  • arguments
  • Math.max(...args)

Остаточные параметры (...)

Если вкратце то: 

            
function argumReturn(a, b, ...arg) {
    let retArg = {}
    retArg.array = arg;
    retArg.lenArray = arg.length
    retArg.sumAB = a+b
    return retArg
}
То есть a, b это параметры, а ...arg это массив, остаточных параметров. Может быть вообще один: 
            
function sumAll(...args) { // args — имя массива
  let sum = 0;

  for (let arg of args) sum += arg;

  return sum;
}

Остаточные параметры должны располагаться в конце

Иначе будет ошибка.

Переменная "arguments"

Ф-ция может быть вообще без параметров, а все параметры собираются в специальный массив: 

    
function args(){
    console.log(arguments);
    console.log(arguments.length)
}

Стрелочные функции не имеют "arguments"

Если мы обратимся к arguments из стрелочной функции, то получим аргументы внешней «нормальной» функции 
    
function f() {
  let showArg = () => alert(arguments[0]);
  showArg(2);
}

f(1); // 1

Оператор расширения

            
alert( Math.max(3, 5, 1) ); // 5
let arr = [3, 5, 1];

alert( Math.max(arr) ); // NaN
Чтобы обойти это ограничение придумали расширение ... 
            
let arr = [3, 5, 1];

alert( Math.max(...arr) );
Массивов может быть несколько, и они могут комбинироваться с обычными переменными: 
            
let arr1 = [1, -2, 3, 4];
let arr2 = [8, 3, -8, 1];

alert( Math.max(1, ...arr1, 2, ...arr2, 25) ); // 2
Так же это можно использовать для слияния массивов: 
            
let arr = [3, 5, 1];
let arr2 = [8, 9, 15];

let merged = [0, ...arr, 2, ...arr2];

alert(merged); // 0,3,5,1,2,8,9,15 (0, затем arr, затем 2, в конце arr2)
Разбития строк на отдельные буквы и помещения этого в массив: 
            
let str = "Привет";

alert( [...str] ); // П,р,и,в,е,т
Результат похожий на работу Array.from(): 
        
let str = "Привет";

// Array.from преобразует перебираемый объект в массив
alert( Array.from(str) ); // П,р,и,в,е,т
Но между Array.from(obj) и [...obj] есть разница:
  • Array.from работает как с псевдомассивами, так и с итерируемыми объектами
  • Оператор расширения работает только с итерируемыми объектами
Выходит, что если нужно сделать из чего угодно массив, то Array.from — более универсальный метод.

Итого

Когда мы видим "..." в коде, это могут быть как остаточные параметры, так и оператор расширения. Как отличить их друг от друга:

  • Если ... располагается в конце списка параметров функции, то это «остаточные параметры». Он собирает остальные неуказанные аргументы и делает из них массив.
  • Если ... встретился в вызове функции или где-либо ещё, то это «оператор расширения». Он извлекает элементы из массива.
Полезно запомнить:
  • Остаточные параметры используются, чтобы создавать новые функции с неопределённым числом аргументов.
  • С помощью оператора расширения можно вставить массив в функцию, которая по умолчанию работает с обычным списком аргументов.
Вместе эти конструкции помогают легко преобразовывать наборы значений в массивы и обратно. К аргументам функции можно обращаться и по-старому — через псевдомассив arguments.
