Синтаксис "new Function"

Ещё один способ объявления ф-ции

Синтаксис 

                
let func = new Function([arg1, arg2, ...argN], functionBody);
Пример: 
                
let sum = new Function('a', 'b', 'return a + b');
Аргументы к ф-ции могут быть переданы в одной строке через запятую. Ф-ция может быть без аргументов. Стратения применения может быть такая: 
            
let str = ... код, полученный с сервера динамически ...

let func = new Function(str);
func();

Замыкание

Важным отличием такого создания ф-ции является то, что доступ она имеет только к глобальному окружению, и не имеет доступа к родительскому окружению. Так что если надо что-то передавать - в аргументах: 

            
function getFunc() {
  let value = "test";

  let func = new Function('alert(value)');

  return func;
}

getFunc()(); // ошибка: value не определено
Проблема в том, что перед отправкой JavaScript-кода на реальные работающие проекты код сжимается с помощью минификатора – специальной программы, которая уменьшает размер кода, удаляя комментарии, лишние пробелы, и, что самое главное, локальным переменным даются укороченные имена. Если бы new Function имела доступ к внешним переменным, при этом были бы проблемы с минификаторами.

Итого

Синтаксис: 

                
let func = new Function ([arg1, arg2, ...argN], functionBody);
Эти 3 объявления ниже эквивалентны: 
                
new Function('a', 'b', 'return a + b'); // стандартный синтаксис
new Function('a,b', 'return a + b'); // через запятую в одной строке
new Function('a , b', 'return a + b'); // через запятую с пробелами в одной строке
Функции, объявленные через new Function, имеют [[Environment]], ссылающийся на глобальное лексическое окружение, а не на родительское. Поэтому они не могут использовать внешние локальные переменные. Но это очень хорошо, потому что страхует нас от ошибок. Переданные явно параметры – гораздо лучшее архитектурное решение, которое не вызывает проблем у минификаторов.
