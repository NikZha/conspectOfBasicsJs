Прототипное наследование

В разделе речь пойдёт об особенностях наследования в javascript.

[[Prototype]]

Есть два варианта наследования: 


    
            let animal = {
                eats: true
            }
            let dog = {
                bark(){
                   alert("Bark!") 
                }
            }
            dog.__proto__ = animal
            alert(`Dog eat: ${dog.eats}`)
И: 

    
        function testProto2(params) {
            let animal = {
                eats: true
            }
            let dog = {
                food: "meat",
                __proto__: animal
            }
            alert(`Dog eat: ${dog.eats}`)
        }
Надо заметить что __proto__ это геттер/сеттер к свойству объекта [[Prototype]], причём вариант с __proto__ устаревший. JavaScript предполагает, что мы должны использовать функции Object.getPrototypeOf/Object.setPrototypeOf вместо того, чтобы получать/устанавливать прототип. Мы также рассмотрим эти функции позже.

Как понятно из вышесказанного, наследование от одного предка, так же возникает иерархия наследования на вершине который стоит встроенный Object.

Операция записи не использует прототип

Автор приводит функцию из объекта и показывает, что для этого объекта вызывается именно она, а не функция родителя. При чём тут операции записи удаления, когда это обычный полиморфизм: 

            
let animal = {
  eats: true,
  walk() {
    /* этот метод не будет использоваться в rabbit */
  }
};

let rabbit = {
  __proto__: animal
};

rabbit.walk = function() {
  alert("Rabbit! Bounce-bounce!");
};

rabbit.walk(); // Rabbit! Bounce-bounce!
Судя по объяснению дипсика в объекте потомке создаются свойства с именами объекта родителя и они же изменяются, при этом свойства родителя остаются не тронутыми.

Реакция дипсика:
При обращении к свойству, которого нет в объекте-потомке, JavaScript ищет его в прототипе (родителе). Если же мы записываем такое свойство в потомка — создаётся собственное свойство с тем же именем, которое "перекрывает" свойство прототипа. При этом оригинальное свойство родителя остаётся неизменным.»

Короткий вариант для записи:
«Свойства прототипа доступны для чтения, но запись создаёт собственное свойство в потомке, не затрагивая прототип.» 

                
let animal = { eats: true }; // Родитель
let rabbit = { __proto__: animal }; // Потомок

// 1. Чтение (из прототипа)
console.log(rabbit.eats); // true (из animal)

// 2. Запись (создаётся своё свойство)
rabbit.eats = false; 
console.log(rabbit.eats); // false (собственное свойство)
console.log(animal.eats); // true (прототип не изменился)

// 3. Проверка наличия свойства
console.log(rabbit.hasOwnProperty('eats')); // true (теперь eats есть прямо в rabbit)

Значение «this»

При возникновении значения this всегда идёт вызов конкретного объекта из которого возник этот вызов, если из объекта потомка - то вызывается объект потомка.

Дипсик предложил чуть другую формулировку: this всегда ссылается на объект, из которого был вызван метод, независимо от того, где этот метод объявлен (в родителе или потомке).

Цикл for…in

Цикл проходит по собственным свойствам объекта и по унаследованным (с учётом, того, что если унаследованное св-во изменяется, оно становится своим).

Свои и чужие свойства и методы можно отфильтровать при помощи встроенного метода obj.hasOwnProperty(key): он возвращает true, если у obj есть собственное, не унаследованное, свойство с именем key.

Свойства появляются в цикле если их значение enumerable - true

Итого

  • В JavaScript все объекты имеют скрытое свойство [[Prototype]], которое является либо другим объектом, либо null.
  • Мы можем использовать obj.__proto__ для доступа к нему (исторически обусловленный геттер/сеттер, есть другие способы, которые скоро будут рассмотрены).
  • Объект, на который ссылается [[Prototype]], называется «прототипом».
  • Если мы хотим прочитать свойство obj или вызвать метод, которого не существует у obj, тогда JavaScript попытается найти его в прототипе.
  • Операции записи/удаления работают непосредственно с объектом, они не используют прототип (если это обычное свойство, а не сеттер).
  • Если мы вызываем obj.method(), а метод при этом взят из прототипа, то this всё равно ссылается на obj. Таким образом, методы всегда работают с текущим объектом, даже если они наследуются.
  • Цикл for..in перебирает как свои, так и унаследованные свойства. Остальные методы получения ключей/значений работают только с собственными свойствами объекта.
