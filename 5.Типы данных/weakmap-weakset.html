WeakMap и WeakSet

Эти две структуры созданы для того, чтобы не увеличивать колличество ссылок которые подсчитывает сборщик мусора. При удалении объекта объект из этих струтур так же пропадает.

WeakMap

Ключи WeakMap должны быть объектами. Небольшая проработка дана тут в ходе неё было выяснено, что ссылка исчезает из этой струтуры очень быстро, буквально секунды. 

                
let john = { name: "John" };

let weakMap = new WeakMap();
weakMap.set(john, "...");

john = null; /
В WeakMap присутствуют только следующие методы:
  • weakMap.get(key)
  • weakMap.set(key, value)
  • weakMap.delete(key)
  • weakMap.has(key)

Пример: дополнительные данные

В этом разделе рассказано основное применение WeakMap - как средство организации слабого хранилица, из которого пропадают объекты по мере их исчезновения где-то ещё. Дипсик говорит что основное применение полуавтоматическое управление памяти, для того, чтобы избежать накопления мусора.

Применение для кеширования

Применяется для кеширования каких-либо действий в виде: 

                    
// 📁 cache.js
let cache = new WeakMap();

// вычисляем и запоминаем результат
function process(obj) {
  if (!cache.has(obj)) {
    let result = /* вычисляем результат для объекта */ obj;

    cache.set(obj, result);
  }

  return cache.get(obj);
}

// 📁 main.js
let obj = {/* какой-то объект */};

let result1 = process(obj);
let result2 = process(obj);

// ...позже, когда объект больше не нужен:
obj = null;

// Нет возможности получить cache.size, так как это WeakMap,
// но он равен 0 или скоро будет равен 0
// Когда сборщик мусора удаляет obj, связанные с ним данные из кеша тоже удаляются
При использовании Map объект кеширования остался бы в памяти.

WeakSet

Коллекция WeakSet ведёт себя похоже:

  • Она аналогична Set, но мы можем добавлять в WeakSet только объекты (не примитивные значения).
  • Объект присутствует в множестве только до тех пор, пока доступен где-то ещё.
  • Как и Set, она поддерживает add, has и delete, но не size, keys() и не является перебираемой.
Нужна условно для того, чтобы определить было ли событие в текущей сессии.
