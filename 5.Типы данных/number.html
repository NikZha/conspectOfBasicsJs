Числа

В JS есть два вида чисел:

  1. Сразу double который иногда прикидывается интом. Но по факту это double 64 разрядное число с плавающей точкой.
  2. BigInt - целое число произвольной длинны.

Тут будет рассмотрено только первое.

Способы записи числа

Способы записи числа:

  • Самый очевидный: 
        
let billion = 1000000000;
  • Метод с синтаксическим сахаром: 
                        
let billion = 1_000_000_000

    Разделяя разряды для понимания.

  • Сокращённая запись eX: 
        
let billion = 1e9;  // 1 миллиард, буквально: 1 и 9 нулей
1e3 === 1 * 1000 // e3 означает *1000
1.23e6 === 1.23 * 1000000 // e6 означает *1000000
  • Тоже самое но разряды от запятой: 
                        
let ms = 1e-6; // шесть нулей слева от 1 тоже самое что и 0.000001

Шестнадцатеричные, двоичные и восьмеричные числа

Способы записи шестнадцатеричных чисел:

            
alert( 0xff ); // 255

Способы записи двоичных и восьмиричных чисел:

            
let a = 0b11111111; // двоичная (бинарная) форма записи числа 255
let b = 0o377; // восьмеричная форма записи числа 255

toString(base)

Метод num.toString(base) возвращает строковое представление числа num в системе счисления base. Например:

                
let num = 255;

alert( num.toString(16) );  // ff
alert( num.toString(2) );   // 11111111

Округление

В JS ест несколько ф-ций округления:

  • Math.floor
    Округление в меньшую сторону: 3.1 становится 3, а -1.1 — -2.
  • Math.ceil
    Округление в большую сторону: 3.1 становится 4, а -1.1 — -1.
  • Math.round
    Округление до ближайшего целого: 3.1 становится 3, 3.6 — 4, а -1.1 — -1.
  • Math.trunc
    Производит удаление дробной части без округления: 3.1 становится 3, а -1.1 — -1.

Неточные вычисления

В силу того, что Number это double и его целая часть хранится отдельно от десятичной происходит для проблем:

  • Бесконечности (переполнения) 
                            
alert( 1e500 ); // Infinity
  • Потери точности: 
                            
alert( 0.1 + 0.2 == 0.3 ); // false
alert( 0.1 + 0.2 ); // 0.30000000000000004
alert( 0.1.toFixed(20) ); // 0.10000000000000000555

Предлагаемое автором учебника решение использовать:

                
let sum = 0.1 + 0.2;
alert( sum.toFixed(2) ); // "0.30"
lert( +sum.toFixed(2) ); // 0.3

Проверка: isFinite и isNaN

  • NaN - ошибка.
  • Infinity и -Infinity бесконечность.

Для проверки NaN используют:

            
alert( isNaN(NaN) ); // true
alert( isNaN("str") ); // true
alert( NaN === NaN ); // false то есть в прямую проверить нельзя. Только ф-цией.

Тоже самое касается и проверки isFinite(): Она возвращает true если это обычное число и false если это не число или бесконечность:

                
alert( isFinite("15") ); // true
alert( isFinite("str") ); // false, потому что специальное значение: NaN
alert( isFinite(Infinity) ); // false, потому что специальное значение: Infinity

Так же есть отдельно методы Number.isNaN и Number.isFinite они более строгие и работают чуть по другому.

Так же для проверки равенства двух значений существует метод Object.is он работает:

  • с NaN: Object.is(NaN, NaN) === true, здесь он хорош.
  • Значения 0 и -0 разные: Object.is(0, -0) === false, это редко используется, но технически эти значения разные.

Во всех других случаях Object.is(a, b) идентичен a === b.

parseInt и parseFloat

Используется для вычленения из string чисел. Проблема состоит в том, что если первой будет буква то будет ошибка:

                
alert( parseInt('a123') ); // NaN, на первом символе происходит остановка чтения

А parseIntраспарсит все до точки:

                
alert( parseInt('12.3') ); // 12, вернётся только целая часть

Второй аргумент parseInt(str, radix)

            
alert( parseInt('0xff', 16) ); // 255
alert( parseInt('ff', 16) ); // 255, без 0x тоже работает

alert( parseInt('2n9c', 36) ); // 123456

Другие математические функции

Math.random()
Возвращает псевдослучайное число в диапазоне от 0 (включительно) до 1 (но не включая 1)

Math.max(a, b, c...) / Math.min(a, b, c...)
Возвращает наибольшее/наименьшее число из перечисленных аргументов.

Math.pow(n, power)
Возвращает число n, возведённое в степень power

Задача 5. Случайное число от min до max

Мною решено как:


                
function random(x, y) {
    let randX = Math.random()
    let returnedRandom = randX * y
    while (x > returnedRandom || returnedRandom > y) {
        if (x > returnedRandom) returnedRandom += randX
        if (returnedRandom > y) returnedRandom -= randX
    }
    return returnedRandom
}

Не смотря на то, что эта срань работает, но распределение там далеко от равномерного. Дипсик предлагает следующее:

                
function random(min, max) {
    return Math.random() * (max - min) + min;
