Формат JSON, метод toJSON

JSON как замена toString(), которая автоматически включает в себя все св-ва объекта.

JSON.stringify

JSON (JavaScript Object Notation) – это общий формат для представления значений и объектов. Сейчас это универсальный формат обмена данными

JavaScript предоставляет методы:

  • JSON.stringify для преобразования объектов в JSON.
  • JSON.parse для преобразования JSON обратно в объект.
Например: 
        
let obj = {
    name: "Peter",
    age: 30,
    isAdmin: true,
    courses: ['html', 'css', 'js'],
    wife: null
}

let str = JSON.stringify(obj);

console.log(`type str: ${typeof str}\n type obj: ${typeof obj}\n str is: ${str}`)
Полученная строка str называется JSON-форматированным или сериализованным объектом. Мы можем отправить его по сети или поместить в обычное хранилище данных.

Отличия от объекта:

  • Строки используют двойные кавычки. Никаких одинарных кавычек или обратных кавычек в JSON. Так 'John' становится "John".
  • Имена свойств объекта также заключаются в двойные кавычки. Это обязательно. Так age:30 становится "age":30.

JSON поддерживает следующие типы данных:

  • Объекты { ... }
  • Массивы [ ... ]
  • Примитивы: строки, числа, логические значения false/true, null.
Например: 
    
// число в JSON остаётся числом
alert( JSON.stringify(1) ) // 1

// строка в JSON по-прежнему остаётся строкой, но в двойных кавычках
alert( JSON.stringify('test') ) // "test"

alert( JSON.stringify(true) ); // true

alert( JSON.stringify([1, 2, 3]) ); // [1,2,3]

JSON является независимой от языка спецификацией для данных, поэтому JSON.stringify пропускает некоторые специфические свойства объектов JavaScript:

  • Свойства-функции (методы).
  • Символьные ключи и значения.
  • Свойства, содержащие undefined. 
        
let user = {
  sayHi() { // будет пропущено
    alert("Hello");
  },
  [Symbol("id")]: 123, // также будет пропущено
  something: undefined // как и это - пропущено
};

alert( JSON.stringify(user) ); // {} (пустой объект)

Самое замечательное, что вложенные объекты поддерживаются и конвертируются автоматически. 

    
let meetup = {
  title: "Conference",
  room: {
    number: 23,
    participants: ["john", "ann"]
  }
};

alert( JSON.stringify(meetup) );
/* вся структура преобразована в строку:
{
  "title":"Conference",
  "room":{"number":23,"participants":["john","ann"]},
}
*/
Важное ограничение: не должно быть циклических ссылок. 
    
let room = {
  number: 23
};

let meetup = {
  title: "Conference",
  participants: ["john", "ann"]
};

meetup.place = room;       // meetup ссылается на room
room.occupiedBy = meetup; // room ссылается на meetup

JSON.stringify(meetup); // Ошибка: Преобразование цикличной структуры в JSON
В целом я вижу решение проблемы в создании копии объекта путём: 
    
meetup.place = Object.fromEntries(Object.entries(room));
Но дипсик утверждает, что это не решение проблемы. Посмотрим что там во второй задаче предлагает автор.

Исключаем и преобразуем: replacer

Полная ф-ция JSON.stringify: 

            
let json = JSON.stringify(value[, replacer, space])
где, value - это объект, replacer это ф-ция для кодирования или массив, space отступы для форматирования в колличественном выражении.

В этом примере автор предлагает обойти свойства которые являются перекрёстными ссылками: 

            
let room = {
  number: 23
};

let meetup = {
  title: "Conference",
  participants: [{name: "John"}, {name: "Alice"}],
  place: room // meetup ссылается на room
};

room.occupiedBy = meetup; // room ссылается на meetup

alert( JSON.stringify(meetup, ['title', 'participants']) );
Но потом начинает торговаться, что хотелось бы побольше св-в, и так приходит к написанию нормальной ф-ции которая оставляет св-во undefined: 
            
let room = {
  number: 23
};

let meetup = {
  title: "Conference",
  participants: [{name: "John"}, {name: "Alice"}],
  place: room // meetup ссылается на room
};

room.occupiedBy = meetup; // room ссылается на meetup

alert( JSON.stringify(meetup, function replacer(key, value) {
  alert(`${key}: ${value}`);
  return (key == 'occupiedBy') ? undefined : value;
}));
Причём реализация сразу наивная, по названию св-ва мы прописывает undefined.

Форматирование: space

Третий параметр в ф-ции JSON.stringify(obj) это параметр форматирующих пробелов. Применяется он для логгирования и красивого вывода

Пользовательский «toJSON»

            
let room = {
  number: 23,
  toJSON() {
    return this.number;
  }
};

let meetup = {
  title: "Conference",
  room
};
Теперь вывод будет следующим: 
            
alert( JSON.stringify(room) ); // 23

alert( JSON.stringify(meetup) );
/*
  {
    "title":"Conference",
    "room": 23
  }
*/

JSON.parse

Перевод JSON to obj. Синтаксис: 

            
let value = JSON.parse(str[, reviver]);
где str - JSON, a reviver необязательная ф-ция которая будет вызываться для каждой пары ключ-значение.

Использование reviver

    
let str = '{"title":"Conference","date":"2017-11-30T12:00:00.000Z"}';

 meetup = JSON.parse(str);

// console.log(meetup.date.getDate()) //Ошибка так как дата в текстовом формате

meetup = JSON.parse(str, function(key, value) {
  if (key == 'date') return new Date(value);
  return value;
});

console.log(meetup.date.getDate())
Вся суть этой ф-ции, обрабатывать пару ключ-значение. Работает и для вложенных значений.

Итого

  • JSON – это формат данных, который имеет собственный независимый стандарт и библиотеки для большинства языков программирования.
  • JSON поддерживает простые объекты, массивы, строки, числа, логические значения и null.
  • JavaScript предоставляет методы JSON.stringify для сериализации в JSON и JSON.parse для чтения из JSON.
  • Оба метода поддерживают функции преобразования для интеллектуального чтения/записи.
  • Если объект имеет метод toJSON, то он вызывается через JSON.stringify.
