Флаги и дескрипторы свойств

К свойствам объекта можно установить флаги и дескрипторы. Об этом речь пойдёт в этой главе.

Флаги свойств

Как уже было написано выше помимо значения (value) свойства есть флаги его определяющие это:

  • writable – если true, свойство можно изменить, иначе оно только для чтения.
  • enumerable – если true, свойство перечисляется в циклах, в противном случае циклы его игнорируют.
  • configurable – если true, свойство можно удалить, а эти атрибуты можно изменять, иначе этого делать нельзя
Посмотреть и потестить код 
    
                let obj = {
                    id: 1,
                    name: "Jo Baden"
                }
                let decr = Object.getOwnPropertyDescriptor(obj, 'name')
                alert(JSON.stringify(decr))
Как видно из примера - дескриптор возвращается в качестве объекта с пакетом этих свойств и установленных к ним значений

Чтобы изменить значения свойств, используется метод Object.defineProperty. Его синтаксис: 

    
Object.defineProperty(obj, propertyName, descriptor)
В качестве примера: 
    
                let obj = {
                    id: 1,
                    name: "Jo Baden"
                }
                alert(`Объект до: ${JSON.stringify(obj)}`)
                Object.defineProperty(obj, "name", {value: "Test"});
                let decr = Object.getOwnPropertyDescriptor(obj, 'name')
                alert(`Объект после: ${JSON.stringify(obj)}`)
                alert(`Флаги свойств: ${JSON.stringify(decr)}`)
Однако стоит заметить, что если задавать с нуля свойство то значения его свойств по умолчанию будет false: 
    
        let obj ={};
        Object.defineProperty(obj, "name", {value: "Barak Obmana"});
        let descr = Object.getOwnPropertyDescriptor(obj, "name")
        alert(`Properties afet:\n${JSON.stringify(descr)}`)
Тут следует отметить, что есть два метода Object.defineProperty и Object.defineProperties. Через последнее не задаётся одно свойство. Потратил на это полчаса.

Только для чтения

Как было показано выше свойство можно сделать доступным только для чтения: 

    
             
            let obj = {name: "John"}
            Object.defineProperty(obj, "name", {writable: false})
            alert(`Пытаемся obj.name = "Pete"${obj.name = "Pete"}\nПолучаем: ${obj.name}`)
Автор пишет что получаем мы ошибки только в строгом режиме use strict и это действительно так: с этой строкой скрипт не запускается, а падает в ошибку. Как было написано выше - свойство изначально false при создании их Object.defineProperty(), даже если какие-то свойства устанновить в true, другие будут по умолчанию в false.

Неперечислимое свойство

Иногда возникает необходимость отключить итерирование свойства: 

  
                
                let usr = {
                    name: "Ivan",
                    toString() {
                        return this.name
                    }
                }
                for (const key in usr) alert(key);
                Object.defineProperty(usr, "toString", {enumerable: false})
                for (const key in usr) alert(key);

Неперечислимые свойства также не возвращаются Object.keys: 
           
alert(Object.keys(usr)); // name

Неконфигурируемое свойство

Флаг неконфигурируемого свойства (configurable:false) иногда предустановлен для некоторых встроенных объектов и свойств. Неконфигурируемое свойство не может быть удалено, его атрибуты не могут быть изменены. Например, свойство Math.PI – только для чтения, неперечислимое и неконфигурируемое: 

            
let descriptor = Object.getOwnPropertyDescriptor(Math, 'PI');

alert( JSON.stringify(descriptor, null, 2 ) );
Math.PI = 3; // Ошибка, потому что writable: false

// delete Math.PI тоже не сработает
Изменить тоже нельзя: 
    
// Ошибка, из-за configurable: false
Object.defineProperty(Math, "PI", { writable: true });
Определение свойства как неконфигурируемого – это дорога в один конец. Мы не можем изменить его обратно с помощью defineProperty.
Обратите внимание: configurable: false не даст изменить флаги свойства, а также не даст его удалить. При этом можно изменить значение свойства. можно изменить если изначально так сконфигурировать: 
    
let user = {
  name: "John"
};

Object.defineProperty(user, "name", {
  configurable: false
});

user.name = "Pete"; // работает
delete user.name; // Ошибка
То есть в данном примере изначально было writable: true, оно же им же и осталось. Если изменить вместо с: 
    
let user = {
  name: "John"
};

Object.defineProperty(user, "name", {
  writable: false,
  configurable: false
});
То свойство останется навечно запечатанным как Math.PI

Метод Object.defineProperties

Существует метод Object.defineProperties(obj, descriptors), который позволяет определять множество свойств сразу. 

    
Object.defineProperties(obj, {
  prop1: descriptor1,
  prop2: descriptor2
  // ...
})
Пример: 
    
Object.defineProperties(user, {
  name: { value: "John", writable: false },
  surname: { value: "Smith", writable: false },
  // ...
});
Таким образом определяется сразу несколько свойств одной операцией.

Object.getOwnPropertyDescriptors

Получит все свойства сразу можно при помощи Object.getOwnPropertyDescriptors(obj) На этом основании можно сделать копирование объекта: 

            
let clone = Object.defineProperties({}, Object.getOwnPropertyDescriptors(obj));
Особенностью данного метода является то, что он возвращает все флаги всех свойств, а так же те свойства который не войдут в цикл for...in

Глобальное запечатывание объекта

есть методы, которые ограничивают доступ ко всему объекту:

  • Object.preventExtensions(obj)
    Запрещает добавлять новые свойства в объект.
  • Object.seal(obj)
    Запрещает добавлять/удалять свойства. Устанавливает configurable: false для всех существующих свойств.
  • Object.freeze(obj)
    Запрещает добавлять/удалять/изменять свойства. Устанавливает configurable: false, writable: false для всех существующих свойств.
А также есть методы для их проверки:
  • Object.isExtensible(obj)
    Возвращает false, если добавление свойств запрещено, иначе true.
  • Object.isSealed(obj)
    Возвращает true, если добавление/удаление свойств запрещено и для всех существующих свойств установлено configurable: false.
  • Object.isFrozen(obj)
    Возвращает true, если добавление/удаление/изменение свойств запрещено, и для всех текущих свойств установлено configurable: false, writable: false.
Но эти методы применяются редко.
