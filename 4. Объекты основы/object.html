Объекты

Ещё один тип данных которых объединяет разные типы данных в виде свойств объекта. Ситаксис создания:


let user = new Object(); // синтаксис "конструктор объекта"
let user = {};  // синтаксис "литерал объекта"

Литералы и свойства

При использовании литерального синтаксиса можно создавать напрямую свойства объекта:


let user = {     // объект
  name: "John",  // под ключом "name" хранится значение "John"
  age: 30        // под ключом "age" хранится значение 30
};

Вызываются свойства путём обращения к ним через точку:


// получаем свойства объекта:
alert( user.name ); // John
alert( user.age ); // 30

Можно добавлять новые свойства так же через точку:


user.isAdmin = true;

Свойства так же легко удаляются:


delete user.age;

Если свойство состоит из нескольких слов оно должно быть закавычено:

        
let user = {
  name: "John",
  age: 30,
  "likes birds": true  // имя свойства из нескольких слов должно быть в кавычках
};

Последнее слово объекта тоже может оканчиваться запятой, что упрощает добавление свойств:

            
let user = {
  name: "John",
  age: 30,
}

Объект созданый при помощи const может быть изменён:

            
const user = {
  name: "John"
};

user.name = "Pete"; // (*)

alert(user.name); // Pete

const тут служит для того, чтобы переменная не переопределялась.

Квадратные скобки

По аналогии с массивом можно использовать квадратные скобки для доступа к свойствам объекта. Это более универсальный способ:

                
let user = {};
user["likes birds"] = true;
alert(user["likes birds"]); // true

let key = "likes birds";
// то же самое, что и user["likes birds"] = true;
user[key] = true;

Так же в квадратных скобках можно использовать переменные:

                
let user = {
  name: "John",
  age: 30
};
let key = prompt("Что вы хотите узнать о пользователе?", "name");
// доступ к свойству через переменную
alert( user[key] ); // John (если ввели "name")
//Но:
let key = "name";
alert( user.key ); // undefined

Вычисляемые свойства

Мы можем использовать квадратные скобки в литеральной нотации для создания вычисляемого свойства:


            
let fruit = prompt("Какой фрукт купить?", "apple");

let bag = {
    [fruit]: 5,
};
for (key in bag){
    alert(`Свойство объекта bag ${key} имеет значение: ${bag[key]}`)
}

А можно прямо напрямую:

            
let fruit = prompt("Какой фрукт купить?", "apple");
let bag = {};

// имя свойства будет взято из переменной fruit
bag[fruit] = 5;    

let fruit = 'apple';
let bag = {
  [fruit + 'Computers']: 5 // bag.appleComputers = 5
};

Таким образом квадратные скобки дают больше возможностей по манипулированию свойствами.

Свойство из переменной

Можно создавать свойство объекта из переменной:

            
function makeUser(name, age) {
  return {
    name: name,
    age: age
    // ...другие свойства
  };
}

let user = makeUser("John", 30);
alert(user.name); // John

Так же все это можно осуществить проще:


            
function makeUser(name, age) {
    return {
        name,
        age,
    };
}
let user = makeUser('John', 35)
for(key in user){
    alert(`Название свойства: ${key} значение свойства ${user[key]}`)
}

Таким образом свойством становиться переменная, а её значением значение переменной.


            
let user = {
  name,  // тоже самое, что и name:name
  age: 30
};

Ограничения на имена свойств

Ограничения на имена свойств объектов не связаны с ограничения на именна переменных js:


            
let obj = {
  for: 1,
  let: 2,
  return: 3,
  0: 4,
};

Однако есть такие свойства объекта которые невозможно изменить:

            
let obj = {};
obj.__proto__ = 5; // присвоим число
alert(obj.__proto__); // [object Object], значение - это объект, т.е. не то, что мы ожидали
