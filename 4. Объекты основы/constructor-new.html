Конструктор, оператор "new"

Этот оператор позволяет создават объект.

Функция-конструктор

Есть ряд соглашений по конструированию объектов функциями:

  • Имя функции-конструктора должно начинаться с большой буквы.
  • Функция-конструктор должна выполняться только с помощью оператора "new".

Происходит примерно следующее:

            
let user = new User("Jack");

function User(name) {
  // this = {};  (неявно)

  // добавляет свойства к this
  this.name = name;
  this.isAdmin = false;

  // return this;  (неявно)
}

Проверка на вызов в режиме конструктора: new.target

Это проверка и запрет вызова ф-ции без new:

            
function User(name) {
  if (!new.target) { // в случае, если вы вызвали меня без оператора new
    return new User(name); // ...я добавлю new за вас
  }

  this.name = name;
}

let john = User("John"); // переадресовывает вызов на new User
alert(john.name); // John

Но в целом зачем и кому это понадобится остаётся вопросом - надо или использовать слово class или фабрику объектов, на мой взгляд. Основная проблема многого что изложено в учебнике - нет разграничения что из ES6, а что легаси возможности. На мой взгляд лучше пользоваться ES6.

Возврат значения из конструктора, return

Обычно то что называет автор констукторами - не требуют return, но если оно там есть то применяются следующие правила:

  • При вызове return с объектом, вместо this вернётся объект.
  • При вызове return с примитивным значением, оно проигнорируется.

Другими словами, return с объектом возвращает этот объект, во всех остальных случаях возвращается this. Например: 

            
function BigUser() {

  this.name = "John";

  return { name: "Godzilla" };  // >-- возвращает этот объект
}

Ещё один момент, скобки желательны, но не требуюся:

            
let user = new User; // >-- без скобок
// то же, что и
let user = new User();

Создание методов в конструкторе

В целом, то что автор говорит о констукторах какая-то дичь, непонятно зачем и где это нужно, проще сделать фабрику создания объектов:

            
function makeUser(name, age, isAdmin) {
    return {
        name,
        age,
        isAdmin,
        greet() {
            return `Hi ${name}! You age is ${age}. You are admin is ${isAdmin}!`
        }
    }
}
let john = makeUser('John', 35, false)
alert(john.greet())

И как мне кажется что это сильно проще, чем писать нечто подобное:

            
function User(name) {
  this.name = name;

  this.sayHi = function() {
    alert( "Меня зовут: " + this.name );
  };
}

Потому что это многословнее, это вносит путаницу этим this.sayHi = function() {...} потому что отличается от нотации JSON. А для сложных объектов есть ключевое слово class.

Показал конспект deepseek, он отметил пару возможностей: 

                
function User(name) {
  this.name = name;
}
// Метод добавляется в прототип и будет общим для всех объектов
User.prototype.greet = function() {
  return `Hi ${this.name}!`;
};

В случае prototype эта фукнция будет разделяться между объектами. Что может быть критично если экземляров объектов много.

Вторая возможность это проверка instanceof на то является ли объект экземляром данной ф-ции.

Итого

Ф-ция конструктор нужна в очень ограниченном числе случаев - по каким-то причинам class избыточен, но при этом самих объекто можно быть много. Во всех остальных случаях - фабрики и class.
