Копирование объектов и ссылки

Вкратце все что нужно знать: примитивные переменные копируются по значению, а объекты по ссылке. Под объектами понимаются: объекты, массивы, ф-ции, встроенные объекты типа дат и т.д.

Сравнение по ссылке

При сравнении объектов сравнивается не значение объекта, а ссылка. Если она одна и та же то true, если разная то false.

Клонирование и объединение, Object.assign

Помимо очевидной самописной фукнции клонировать и объединять объекты призвана библиотечная ф-ция:

Object.assign(dest, [src1, src2, src3...]) где:

  • dest - объект в который копируют.
  • scr1 ... scrN - объекты которые копируются в dest

Так же эта встроенная статическая ф-ция возвращает dest, что позволяет сделать так:

            
let user = {
  name: "John",
  age: 30
};

let clone = Object.assign({}, user);

Ещё одна встроенная возможность клонирования объекта:

            
clone = {...user}

Вложенное клонирование

Если объект имеет объекты как свойства, то клонирование приводит к копированию ссылок на эти объекты. Чтобы клонировать и эти объекты можно использовать ф-цию _.cloneDeep(obj) из библиотеки lodash. В современных браузерах есть ф-ция structuredClone().
