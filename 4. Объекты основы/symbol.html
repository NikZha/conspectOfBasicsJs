Тип данных Symbol

По спецификации, в качестве ключей для свойств объекта могут использоваться только строки или символы. Ни числа, ни логические значения не подходят, разрешены только эти два типа данных.

Символы

Каждый символ уникален:

            
let id = Symbol();
// Создаём символ id с описанием (именем) "id"
let id = Symbol("id");
//Уникальность символов:
let id1 = Symbol("id");
let id2 = Symbol("id");

alert(id1 == id2); // false

Символы автоматически не преобразуются в строки:

            
let id = Symbol("id");
alert(id); // TypeError: Cannot convert a Symbol value to a string

Но при этом обладают свойствами для преобразования:

            
alert(id.toString()); // Symbol(id), теперь работает
alert(id.description); // id

«Скрытые» свойства

Смысл в том, что класс Symbol позволяет создавать уникальный идентификатор и только по этому идентификатору, можно обратиться к свойству объекта:

                
let user = {
  name: "Вася"
};

let id = Symbol("id");

user[id] = 1;

alert( user[id] ); // мы можем получить доступ к данным по ключу-символу

Символы в литеральном объекте

Объявление символьного объекта в литеральном объекте:

            
let id = Symbol("id");

let user = {
  name: "Вася",
  [id]: 123 // просто "id: 123" не сработает
};

Символы игнорируются циклом for…in


            
let id = Symbol("id");
let user = {
  name: "Вася",
  age: 30,
  [id]: 123
};

for (let key in user) alert(key);

Это – часть общего принципа «сокрытия символьных свойств». А вот Object.assign, в отличие от цикла for..in, копирует и строковые, и символьные свойства: 

            
let id = Symbol("id");
let user = {
  [id]: 123
};

let clone = Object.assign({}, user);

alert( clone[id] ); // 123

Однако получить доступ из цикла ко всем ключам возможно:


    
const allKeys = Reflect.ownKeys(user);
for (let key of allKeys) {
    console.log(key, user[key]); 
}
