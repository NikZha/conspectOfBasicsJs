Переменные

Переменные нужны для хранения информации.

Переменная

Переменная это именнованное хранилище данных. Может инициализироваться сразу или отложенно, так же можно объявить несколько переменных в одной строке, только это плохая практика.


let message;

message = 'Hello';

let message2 = 'Hello!';      
//Плохая практика:
let user = 'John', age = 25, message = 'Hello';

Переменная может быть объявлена при помощи ключевого слова var. Важно понимать:

  • Если объявить var вне любой функции — переменная станет глобальной.
  • Если объявить внутри функции — она будет доступна только внутри этой функции.
  • В отличие от let/const, var игнорирует блочную область видимости (if, for, while).


// 1. Глобальная var
var global = "я глобальная";
console.log(window.global); // "я глобальная"

// 2. Функциональная var
function example() {
    var local = "я локальная";
    console.log(local); // "я локальная"
}
example();
console.log(local); // ❌ Ошибка!

// 3. var игнорирует блоки
if (true) {
    var blockVar = "я видна снаружи!";
}
console.log(blockVar); // "я видна снаружи!"

В принципе лучше просто так var не пользоваться. Это пережиток прошлого существующий только из-за необходимости обратной совместимости.

Аналогия из жизни

Автор приводит аналогию с коробкой, сообщая что если присвоить другое значение, то предыдущее пропадёт. Так же сообщает, что переобъявление переменной вызовет ошибку. В js где динамическая типизация этого и не требуется.

Имена переменных

В JavaScript есть два ограничения, касающиеся имён переменных:

  1. Имя переменной должно содержать только буквы, цифры или символы $ и _.
  2. Первый символ не должен быть цифрой.

В js принят camalCase.

Так же можно использовать не латиницу, но это не рекомендуются в виду слабой читаемости такого. Нельзя использовать ключевые слова в качестве переменных.

Константы

Объявленные константные переменные не возможно изменить, так же нужно объявлять уже с присвоением.


const myBirthday = '18.04.1982';

myBirthday = '01.01.2001'; // ошибка, константу нельзя перезаписать!

Обычно константами объявляются значения которые не будут изменены и иногда заглавными буквами:


const COLOR_RED = "#F00";
const COLOR_GREEN = "#0F0";
const COLOR_BLUE = "#00F";
const COLOR_ORANGE = "#FF7F00";

// ...когда нам нужно выбрать цвет
let color = COLOR_ORANGE;
alert(color); // #FF7F00

Придумывайте правильные имена

Имена в camelCase должны отражать суть переменной. При наличии ide имя любой длинны не является проблемой.

Итого

  • let - современный способ объявления переменных.
  • var - устаревший способ объявления переменных, лучше не пользоваться.
  • const - объявление констант, переменных которые не будут изменяться весь срок работы программы.

Используйте нормальные имена для переменных.
